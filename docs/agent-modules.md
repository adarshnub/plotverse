# Agent Modules

The real-estate mode is represented as a multi-agent workflow. The canonical module list and canvas positions live in `lib/seed.ts`. Execution happens in `lib/agents.ts`.

## Module List

### Source Scout

- Key: `source-scout`
- Model use: none currently.
- Purpose: chooses enabled Kottayam source URLs and run parameters.
- Current output: source run plan for the scraper adapters.

### Scraper Adapters

- Key: `scraper-adapters`
- Model use: none.
- Purpose: runs source-specific Playwright adapters and returns raw public cards.
- Fallback: records per-source failure and lets the rest of the run continue.

### Plot Normalizer

- Key: `plot-normalizer`
- Model use: AI-assisted when `OPENAI_API_KEY` is available.
- Purpose: extracts land-specific facts from raw listing text.
- Fallback: local parser for price, locality, area, unit, sqft, and price per cent.

### Lead Normalizer

- Key: `lead-normalizer`
- Model use: AI-assisted when `OPENAI_API_KEY` is available.
- Purpose: extracts buyer/broker demand from public demand text.
- Contact policy: visible public contact only; no hidden or paid contact extraction.

### Duplicate Resolver

- Key: `duplicate-resolver`
- Model use: none.
- Purpose: flags duplicate source URLs or similar imported plot records.

### Review Gate

- Key: `review-gate`
- Model use: none.
- Purpose: requires approval before scraped records become properties or clients.

### Property Normalizer

- Key: `property-normalizer`
- Model use: none currently.
- Purpose: prepares selected property records for matching.
- Current output: property ids, areas, and prices for the run trace.

### Client Preference Normalizer

- Key: `client-normalizer`
- Model use: none currently.
- Purpose: prepares selected client records for matching.
- Current output: client ids, preferred areas, and max budgets for the run trace.

### Rule Matcher

- Key: `rule-matcher`
- Model use: AI-assisted when `OPENAI_API_KEY` is available.
- Purpose: applies deterministic constraints first, then asks the model to judge nuanced fit.
- Logic source: `lib/matching.ts`.
- Checks include budget, area, property type, bedrooms, size, must-haves, and deal blockers.
- AI function: `aiEvaluateMatch` in `lib/openai.ts`.
- AI can adjust score, pass/fail, reasons, warnings, summary, objections, and next action.
- Fallback: deterministic rule score if `OPENAI_API_KEY` is missing or the model call fails.

### Fit Analyst

- Key: `fit-analyst`
- Model use: shares the AI-assisted evaluation produced during rule matching.
- Function: `aiEvaluateMatch` in `lib/openai.ts`.
- Model selection: `OPENAI_MODEL`, defaulting to `gpt-5`.
- Fallback: deterministic fit summary if `OPENAI_API_KEY` is missing or the call fails.

### Objection Finder

- Key: `objection-finder`
- Model use: currently no separate model call.
- Purpose: records objections generated by deterministic rules and AI enrichment.
- Current output: top warnings across produced matches.

### Outreach Draft Writer

- Key: `draft-writer`
- Model use: OpenAI when a user clicks Draft on a match.
- Function: `draftOutreach` in `lib/openai.ts`.
- Model selection: `OPENAI_MODEL`, defaulting to `gpt-5`.
- Fallback: local message template.
- Important: this module creates drafts only; it does not send messages.

### Approval Gate

- Key: `approval-gate`
- Model use: none.
- Purpose: enforces human approval.
- Current output: audit policy and `outboundActions: 0`.

### Run Reporter

- Key: `run-reporter`
- Model use: none.
- Purpose: stores run summary and trace.
- Output appears in the Runs page and Lab latest trace panel.

## Current Model Policy

All OpenAI-enabled modules share one configurable model:

```ts
OPENAI_MODEL=gpt-5
```

Currently AI-backed modules are:

- Rule Matcher / AI Match Evaluator
- Fit Analyst
- Outreach Draft Writer

If `OPENAI_API_KEY` is missing, runs are marked with `local-rules-fallback` and no model call is made.

## Future Extension Point

To give each module its own model, add a model field to `AgentModule`, store it in `agent_modules`, and pass the selected module model into `aiEvaluateMatch` or `draftOutreach`.
